## Testing Workflow

### Phase 1: Static Analysis
1. Extract APK/IPA: `apktool d app.apk` or unzip IPA
2. Decompile: `jadx -d output app.apk`
3. Review AndroidManifest.xml / Info.plist
4. Search for: hardcoded secrets, API keys, URLs, insecure configurations
5. Check: permissions, exported components, backup settings, debug flags

### Phase 2: Network Analysis
1. Set up proxy (mitmproxy/Burp)
2. Install CA certificate on device
3. Bypass certificate pinning if present
4. Capture all API traffic during app usage
5. Analyze: authentication flows, token handling, data in transit

### Phase 3: Dynamic Analysis
1. Install app on rooted/jailbroken device or emulator
2. Hook with Frida/Objection
3. Test: root detection bypass, SSL pinning bypass
4. Monitor: file system changes, logs, clipboard, screenshots
5. Instrument: crypto functions, auth checks, API calls

### Phase 4: API Security
1. Extract API endpoints from decompiled code
2. Test authentication/authorization
3. Test for IDOR, mass assignment, rate limiting
4. Test input validation on all parameters
5. Check for hidden/debug API endpoints

### Phase 5: Business Logic
1. Test payment flows for tampering
2. Test subscription/license bypass
3. Test referral/reward abuse
4. Test offline functionality security
5. Test data sync between device and server

## Output Format
For each finding:
- **MASVS-ID:** e.g., MASVS-STORAGE-1
- **Title:** e.g., Plaintext credentials in SharedPreferences
- **Severity:** Critical/High/Medium/Low (CVSS score)
- **CWE:** e.g., CWE-312
- **Platform:** Android/iOS/Both
- **Evidence:** Code snippet, screenshot, or traffic capture
- **Impact:** What an attacker could achieve
- **Remediation:** Specific fix recommendations
